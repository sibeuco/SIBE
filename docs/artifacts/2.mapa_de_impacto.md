# Mapa de Impacto — SIBE

---

## Objetivo (Why?)

Para la Dirección de Bienestar y Evangelización que enfrenta problemas con la gestión manual de asistencia a las actividades, SIBE es un aplicativo web que permitirá agendar, organizar y hacer seguimiento a estas actividades de forma eficiente, garantizando la precisión de la información, agilizando los procesos y automatizando la generación de reportes para facilitar auditorías y la toma de decisiones, eliminando procedimientos manuales y el uso de papel, lo que mejorará el desempeño operativo y contribuirá al cuidado del medio ambiente. A diferencia de Google Forms, CheckPoint y AccuClass, nuestro producto podrá gestionar la asistencia a las actividades ingresando el número de identificación del participante o mediante un lector RFID, permitiendo que el proceso se realice de forma rápida y segura con el uso del carnet universitario.

### Actor 2: Administrador de Área

> Usuario con rol `ADMINISTRADOR_AREA`. Gestiona actividades, participantes e indicadores dentro de su área o subáreas asignadas. No tiene acceso a la gestión de usuarios ni a áreas fuera de su ámbito.

| # | Entregable | Trazabilidad |

| 2 | Tabla de actividades del área con filtrado, ordenamiento y acceso a fechas programadas | RF-004 → HU-011 |
| 3 | Edición de actividades existentes incluyendo modificación de fechas programadas pendientes | RF-004 → HU-010 |

#### Impacto 2.2 — Registro digital de asistencia con RFID y documento

> Eliminar las planillas de asistencia en papel mediante un sistema de registro digital que identifica participantes por carnet RFID o número de documento, garantizando precisión y rapidez.

| # | Entregable | Trazabilidad |
| - | ---------- | ------------ |

### Actor 3: Colaborador

> Usuario con rol `COLABORADOR`. Puede consultar actividades y registrar asistencia en las actividades asignadas dentro de su contexto organizacional. No tiene permisos de creación ni eliminación de actividades ni usuarios.

#### Impacto 3.1 — Ejecución ágil del registro de asistencia en campo

> Registrar la asistencia de los participantes durante la ejecución de actividades de forma rápida, utilizando el lector RFID o ingresando el número de documento, sin depender de planillas físicas.

| # | Entregable | Trazabilidad |
| - | ---------- | ------------ |
| 1 | Acceso a la tabla de actividades programadas de su área/subárea para visualizar las fechas disponibles | HU-011 |
| 2 | Iniciar, registrar asistencia y finalizar la ejecución de actividades asignadas | RF-011 → HU-012, HU-013 |
| 3 | Registro de participantes externos (no pertenecientes a la universidad) con documento y nombre completo durante la ejecución de una actividad | RF-011 → HU-012 |
| 4 | Consulta de participantes registrados en una ejecución finalizada | HU-012 |

#### Impacto 3.2 — Gestión autónoma de credenciales

> Cambiar su contraseña de forma autónoma y recuperar acceso al sistema en caso de olvido, sin depender del Administrador.

| # | Entregable | Trazabilidad |
| - | ---------- | ------------ |

### Actor 4: Participante (Estudiante / Empleado / Externo)

#### Impacto 4.1 — Registro de asistencia rápido y sin fricción

> Eliminar la necesidad de firmar planillas físicas, permitiendo que el registro de presencia se realice de forma automática con el carnet universitario o ingresando el documento de identificación.

| # | Entregable | Trazabilidad |
| - | ---------- | ------------ |
| 1 | Identificación automática mediante lectura de carnet RFID (por ID de carnet o número de identificación) | RF-011 → HU-012 |
| 2 | Identificación manual por número de documento para participantes sin carnet disponible | RF-011 → HU-012 |
| 3 | Registro de participantes externos con datos básicos (documento y nombre completo) | RF-011 → HU-012 |
