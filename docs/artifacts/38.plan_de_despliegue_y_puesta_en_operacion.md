## 1. Introducción y Propósito

### 1.1 Propósito

Este artefacto describe el **plan completo de despliegue y puesta en operación** del sistema SIBE mediante **contenedores Docker**. El sistema está preparado para ser clonado, parametrizado y desplegado en cualquier infraestructura que soporte Docker, minimizando ajustes adicionales durante la fase de montaje.

### 1.2 Alcance

| Ambiente | Método de despliegue | Audiencia |
|----------|---------------------|-----------|
| **Desarrollo (local)** | `ng serve` + `./gradlew bootRun` + PostgreSQL local | Desarrolladores |
| **Despliegue Docker (principal)** | `docker compose up --build -d` | Equipo de sistemas / infraestructura |

### 1.3 Contexto

El equipo de sistemas de la organización receptora solicitó:
1. Repositorio Git listo para clonar.
2. Ubicación exacta de la configuración de base de datos.
3. Variables de entorno para PostgreSQL.
4. Mecanismo de creación de tablas y datos iniciales.
5. Dockerfiles y docker-compose con instrucciones de construcción.
6. Dependencias, prerrequisitos y consideraciones técnicas.

Todos estos puntos están implementados y documentados en este artefacto.

                            ├── Capa de Aplicación (CQRS)
                            ├── Dominio (Hexagonal)

## 6. Configuración de Base de Datos

### 6.1 Ubicación de la Configuración

**Archivo:** `SIBEBackend/src/main/resources/application.properties`

```properties
# Data Base (PostgreSQL)
spring.datasource.url=${DB_URL:jdbc:postgresql://localhost:5432/sibe_db2}
spring.datasource.username=${DB_USERNAME:postgres}
spring.datasource.password=${DB_PASSWORD:admin1234}
spring.datasource.driver-class-name=org.postgresql.Driver

# JPA / Hibernate
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
spring.jpa.hibernate.ddl-auto=${DDL_AUTO:update}
```

### 6.2 Comportamiento en Docker

En el despliegue Docker, estas propiedades se sobreescriben automáticamente a través de las variables de entorno definidas en `docker-compose.yml`:

```yaml
environment:
  DB_URL: jdbc:postgresql://db:5432/${POSTGRES_DB:-sibe_db2}
  DB_USERNAME: ${DB_USERNAME:-postgres}
  DB_PASSWORD: ${DB_PASSWORD:-admin1234}
```

El host `db` es el nombre del servicio de PostgreSQL dentro de la red Docker.

## 8. Creación de Tablas y Datos Iniciales

### 8.1 Esquema de Base de Datos

**No se requieren scripts SQL.** El esquema se gestiona automáticamente:

1. **Hibernate** crea/actualiza las ~44 tablas al iniciar la aplicación (`ddl-auto=update`).
2. Los **DataLoaders** insertan automáticamente los datos base del sistema.

### 8.2 DataLoaders (Datos Semilla)

| Orden | DataLoader | Datos cargados |
|-------|-----------|----------------|
| 1 | `TipoUsuarioDataLoader` | 3 tipos: ADMINISTRADOR_DIRECCION, ADMINISTRADOR_AREA, COLABORADOR |
| 2 | `TipoIdentificacionDataLoader` | 3 tipos: CC, TI, CE |
| 3 | `TemporalidadDataLoader` | 5 temporalidades: Diaria, Semanal, Mensual, Trimestral, Anual |
| 4 | `EstadoActividadDataLoader` | 3 estados: PENDIENTE, EN_CURSO, FINALIZADA |
| 5 | `PublicoInteresDataLoader` | 5 públicos de interés |
| 6 | `TipoIndicadorDataLoader` | 5+ tipos de indicador |
| 7 | `SubareaDataLoader` | 8 subáreas de bienestar |
| 8 | `AreaDataLoader` | 4 áreas (con subáreas asociadas) |
| 9 | `DireccionDataLoader` | 1 dirección institucional (con áreas) |
| 10 | `UsuarioAdministradorDataLoader` | 1 usuario administrador por defecto |

**Comportamiento idempotente:** Cada DataLoader verifica si los datos ya existen antes de insertarlos. Es seguro reiniciar la aplicación sin riesgo de duplicación.

## 9. Configuración Docker

### 9.1 Dockerfile — Backend (`SIBEBackend/Dockerfile`)

```dockerfile
# ---- Stage 1: Build ----
FROM gradle:8-jdk17 AS build
WORKDIR /app
COPY build.gradle settings.gradle ./
COPY gradle ./gradle
COPY gradlew ./
RUN gradle dependencies --no-daemon || true
COPY src ./src
RUN gradle bootJar --no-daemon -x test

# ---- Stage 2: Runtime ----
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY --from=build /app/build/libs/*.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 9.2 Dockerfile — Frontend (`SIBEFrontend/Dockerfile`)

```dockerfile
# ---- Stage 1: Build ----
FROM node:18-alpine AS build
WORKDIR /app
COPY package.json package-lock.json* ./
RUN npm ci
COPY . .
RUN npx ng build --configuration production

# ---- Stage 2: Runtime ----
FROM nginx:alpine
COPY --from=build /app/dist/sibe-frontend /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 9.3 Configuración Nginx (`SIBEFrontend/nginx.conf`)

```nginx
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## 10. Procedimiento de Despliegue

### 10.1 Despliegue Inicial (Primera Vez)

```bash
# 1. Clonar el repositorio
git clone <URL_DEL_REPOSITORIO> SIBE
cd SIBE

# 2. (Opcional) Personalizar variables de entorno
cp .env.example .env

# 3. Construir y levantar todos los servicios
docker compose up --build -d
```

**Lo que ocurre automáticamente:**
1. Docker construye las imágenes del backend y frontend (multi-stage builds).
2. PostgreSQL arranca y crea la base de datos `sibe_db2`.
3. El backend espera a que PostgreSQL esté saludable (`healthcheck`).
4. Hibernate crea automáticamente las ~44 tablas del esquema.
5. Los 10 DataLoaders siembran los datos base del sistema.
6. Nginx arranca y sirve la aplicación Angular.

### 10.2 Verificar el Despliegue

```bash
docker compose ps
docker compose logs -f backend
```

## 12. Operación y Mantenimiento

### 12.1 Gestión de Contenedores

```bash
docker compose down                    # Detener servicios
docker compose down -v                 # Detener + borrar datos
docker compose up --build -d backend   # Reconstruir solo backend
docker compose restart backend         # Reiniciar servicio
```

### 12.2 Backup de Base de Datos

```bash
docker compose exec db pg_dump -U postgres sibe_db2 > backup_$(date +%Y%m%d).sql
```

### 12.3 Restaurar Backup

```bash
cat backup.sql | docker compose exec -T db psql -U postgres -d sibe_db2
```

### 12.4 Rollback

```bash
docker compose down
git checkout <commit_anterior>
docker compose up --build -d
```

## 13. Configuración CORS

La configuración CORS del backend es parametrizable mediante la variable `CORS_ALLOWED_ORIGINS`. En `ProjectSecurityConfig.java`, los orígenes se leen dinámicamente desde `app.cors.allowed-origins` en `application.properties`.

Para ajustar en producción:
```
CORS_ALLOWED_ORIGINS=https://sibe.midominio.com
```
