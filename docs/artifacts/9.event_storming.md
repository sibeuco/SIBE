## 1. Introducción

El **Event Storming** es una técnica de taller colaborativo utilizada para descubrir y modelar el flujo de eventos de un dominio de negocio. Para el sistema SIBE, este artefacto documenta los **32 objetos de dominio** identificados durante la sesión de Event Storming, junto con sus comandos, actores, políticas de negocio (reglas), modelos de lectura, eventos de dominio, sistemas externos y las cadenas de eventos previos/posteriores.

El artefacto fue originalmente construido en formato de tablero Excel (draw.io + tarjetas de colores) y en este documento se migra a una estructura de texto limpia, estandarizada y navegable.

**Actores del sistema identificados:**
- **Administrador Dirección** — máximos permisos, gestiona toda la organización
- **Administrador Área** — gestiona su área y sus actividades
- **Colaborador** — ejecuta actividades y registra asistencia
- **Proceso interno del sistema** — acciones automáticas desencadenadas por eventos (carga de datos, primera ejecución)

## 2. Leyenda de Event Storming

| Elemento | Descripción |
|----------|-------------|
| **Evento de Dominio** | Algo que ocurrió en el sistema (pasado, irreversible) |
| **Comando** | Intención que desencadena un evento (imperativo) |
| **Actor** | Persona o sistema que emite el comando |
| **Política de Negocio** | Regla de negocio que debe cumplirse antes de ejecutar el comando |
| **Modelo de Lectura** | Información externa que el actor necesita para emitir el comando |
| **Agregado / Objeto de Dominio** | Entidad central sobre la que actúa el comando |
| **Sistema Externo** | Sistema ajeno al dominio que participa en el flujo |
| **Hotspot** | Aspecto por resolver, riesgo o pregunta abierta |
