## 1. Visión General

Este artefacto documenta los **diagramas de secuencia** principales del sistema SIBE, trazando las interacciones exactas entre los participantes (actores, componentes frontend, capas backend y base de datos) para los flujos más representativos del sistema.

### Convenciones de Notación

| Elemento             | Significado                                                    |
|----------------------|----------------------------------------------------------------|
| Rectángulo sólido    | Participante activo (componente, clase, sistema externo)       |
| Línea continua →     | Mensaje síncrono (llamada directa)                             |
| Línea punteada -->>  | Respuesta / retorno                                            |
| Rectángulo sobre línea | Activación (período de procesamiento)                        |
| `alt` / `else`       | Flujo alternativo (condicional)                                |
| `loop`               | Iteración                                                      |
| `opt`                | Paso opcional                                                  |
| `Note`               | Anotación explicativa                                          |

### Flujos Documentados

| ID    | Nombre                                    | Tipo       | Capas Involucradas                |

## 2. SD-01 — Autenticación (Login Completo)

### 2.1 Descripción

Flujo completo de autenticación desde que el usuario ingresa credenciales en el formulario de login hasta que obtiene un JWT y es redirigido al dashboard. Involucra la cadena de filtros de Spring Security, el proveedor de autenticación, la generación del token JWT, y la persistencia del token en el frontend.

### 2.2 Diagrama de Secuencia

```mermaid
sequenceDiagram
    actor U as Usuario
    participant LC as LoginComponent
    participant LS as LoginService
    participant AI as AuthInterceptor
    participant HC as HttpClient
    participant EF as ExceptionFilter
    participant RVF as RequestValidation<br/>BeforeFilter
    participant ALAT as AuthoritiesLogging<br/>AtFilter
    participant BA as BasicAuth<br/>(Spring Security)
    participant AP as UsernamePwd<br/>AuthenticationProvider
    participant JTGF as JWTTokenGenerator<br/>Filter
    participant ALAF as AuthoritiesLogging<br/>AfterFilter
    participant LCtrl as LoginControlador
    participant LMan as LoginManejador
    participant LUC as LoginUseCase
    participant DAO as UsuarioDAO /<br/>PersonaDAO
    participant DB as PostgreSQL
    participant SS as StateService

    U->>LC: Ingresa correo y clave → click "Iniciar Sesión"
    LC->>LC: validar formularioLogin (email + required)

    alt Formulario inválido
        LC-->>U: Muestra errores de validación
    else Formulario válido
        LC->>LS: validarLogin(Login{correo, contrasena})
        LS->>LS: sessionStorage.setItem('userdetails', JSON.stringify(login))
        LS->>AI: doGet(endpoint + '/login')
        AI->>AI: Lee sessionStorage('userdetails')
        AI->>AI: Construye header Authorization: Basic btoa(correo:contrasena)
        AI->>AI: sessionStorage.removeItem('userdetails')
        AI->>HC: Request con headers Authorization + X-Requested-With

        HC->>EF: HTTP GET /api/login
        EF->>RVF: doFilterInternal(request, response, chain)
        RVF->>ALAT: chain.doFilter()
        Note over ALAT: Log de authorities entrantes (vacías en login)

        ALAT->>BA: chain.doFilter()
        BA->>AP: authenticate(UsernamePasswordAuthenticationToken)
        AP->>DAO: findByCorreo(username)
        DAO->>DB: SELECT * FROM usuario WHERE correo = ?
        DB-->>DAO: UsuarioEntidad
        AP->>AP: encriptarClaveServicio.existe(pwd, user.getClave())

        alt Credenciales inválidas
            AP->>AP: throw AuthorizationException
            EF-->>HC: response.sendError(401, mensaje)
            HC-->>AI: Error 401
            AI-->>LS: Error Observable
            LS-->>LC: Error
            LC-->>U: "Correo o contraseña incorrectos"
        else Credenciales válidas
            AP->>DAO: personaDAO.findByCorreo(username)
            DAO->>DB: SELECT * FROM persona WHERE correo = ?
            DB-->>DAO: PersonaEntidad
            AP->>AP: Construir authorities (ROLE_CODE_READ, CREATE, UPDATE, DELETE)
            AP-->>BA: UsernamePasswordAuthenticationToken + authorities

            BA->>JTGF: chain.doFilter() (post-auth)
            JTGF->>DAO: usuarioDAO.findByCorreo(auth.getName())
            DAO->>DB: SELECT usuario + organizacion
            DB-->>DAO: UsuarioEntidad + UsuarioOrganizacionEntidad
            JTGF->>JTGF: Jwts.builder()<br/>.claim(email, id, authorities, rol,<br/>direccionId, areaId, subareaId)<br/>.setExpiration(now + 30min)<br/>.signWith(HMAC-SHA key)<br/>.compact()
            JTGF->>JTGF: response.setHeader("Authorization", jwt)

            JTGF->>ALAF: chain.doFilter()
            ALAF->>LCtrl: chain.doFilter() → @RequestMapping(/login)
            LCtrl->>LMan: loginManejador.ejecutar(principal.getName())
            LMan->>LUC: loginUseCase.ejecutar(correo)
            LUC->>DAO: personaRepositorioConsulta.consultarUsuarioPorCorreo(correo)
            DAO->>DB: SELECT FROM persona WHERE correo = ?
            DB-->>DAO: Persona con UUID
            LUC-->>LMan: UUID del usuario
            LMan-->>LCtrl: ComandoRespuesta(UUID)
            LCtrl-->>ALAF: ResponseEntity 200 + JSON

            ALAF-->>HC: HTTP 200 + Header Authorization: JWT
            HC-->>AI: Response con header Authorization
            AI->>AI: sessionStorage.setItem('Authorization', jwt)
            AI-->>LS: Observable success
            LS-->>LC: Response(UUID)
            LC->>LC: sessionStorage.getItem('Authorization')
            LC->>LC: Decodificar JWT → UserSession{correo, id, authorities, rol, ...}
            LC->>SS: updateState(USER_SESSION, userSession)
            LC->>LC: router.navigate(['/home'])
            LC-->>U: Redirigido al Dashboard
        end
    end
```

### 2.3 Almacenamiento de Tokens

| Paso | Operación                                    | Storage         |
|------|----------------------------------------------|-----------------|
| 1    | `LoginService` guarda credenciales           | sessionStorage(`userdetails`) |
| 2    | `AuthInterceptor` lee y elimina credenciales | sessionStorage(`userdetails`) → `removeItem` |
| 3    | `AuthInterceptor` captura JWT de respuesta   | sessionStorage(`Authorization`) |
| 4    | `LoginComponent` lee JWT para decodificar    | sessionStorage(`Authorization`) |
| 5    | `StateService` rehidrata sesión desde JWT    | sessionStorage(`Authorization`) |

## 3. SD-02 — Validación de Token JWT en Requests Autenticados

### 3.1 Descripción

Flujo de un request HTTP autenticado desde el frontend, mostrando cómo los interceptores agregan el token y cómo la cadena de filtros del backend lo valida antes de llegar al controlador.

### 3.2 Diagrama de Secuencia

```mermaid
sequenceDiagram
    participant Comp as Angular Component
    participant Svc as Angular Service
    participant AI as AuthInterceptor
    participant TI as TokenInterceptor
    participant HC as HttpClient
    participant EF as ExceptionFilter
    participant RVF as RequestValidation<br/>BeforeFilter
    participant JTVF as JWTTokenValidator<br/>Filter
    participant ALAT as AuthoritiesLogging<br/>AtFilter
    participant Ctrl as @RestController
    participant PA as @PreAuthorize

    Comp->>Svc: método de servicio (e.g. consultarPorArea(id))
    Svc->>AI: HTTP Request
    AI->>AI: Lee sessionStorage('Authorization')<br/>agrega Authorization header
    AI->>AI: Lee sessionStorage('XSRF-TOKEN')<br/>agrega X-XSRF-TOKEN
    AI->>TI: Request con headers
    TI->>TI: cookieService.get('token')

    alt Token en cookie expirado
        TI->>TI: logout(): cookie.delete + sessionStorage.clear + localStorage.clear
        TI-->>Comp: router.navigate(['/login'])
    else Token válido o no hay cookie
        TI->>HC: Request
        HC->>EF: HTTP Request a /api/*

        EF->>RVF: doFilterInternal()
        RVF->>JTVF: chain.doFilter()
        JTVF->>JTVF: request.getHeader("Authorization") → JWT
        JTVF->>JTVF: Jwts.parserBuilder()<br/>.setSigningKey(HMAC key)<br/>.parseClaimsJws(jwt)

        alt JWT inválido o expirado
            JTVF->>JTVF: throw BadCredentialsException
            EF-->>HC: response.sendError(400, "Token inválido")
            HC-->>TI: Error 400/401
            TI->>TI: logout()
            TI-->>Comp: router.navigate(['/login'])
        else JWT válido
            JTVF->>JTVF: Extraer claims: email, authorities,<br/>rol, direccionId, areaId, subareaId
            JTVF->>JTVF: Construir ContextoUsuarioAutenticado
            JTVF->>JTVF: SecurityContextHolder.setAuthentication(auth)
            JTVF->>ALAT: chain.doFilter()
            Note over ALAT: Log authorities del usuario autenticado
            ALAT->>Ctrl: chain.doFilter() → Método del controlador
            Ctrl->>PA: @PreAuthorize verifica rol/authority

            alt Sin permiso suficiente
                PA-->>EF: AccessDeniedException
                EF-->>HC: response.sendError(403)
                HC-->>Comp: Error 403
            else Permiso OK
                Ctrl->>Ctrl: Ejecutar lógica del endpoint
                Ctrl-->>HC: ResponseEntity 200 + JSON
                HC-->>AI: Response
                AI-->>Svc: Observable con datos
                Svc-->>Comp: Datos tipados
            end
        end
    end
```

## 4. SD-03 — Crear Actividad (Flujo de Comando/Escritura)

### 4.1 Descripción

Flujo CQRS de escritura completo: desde la UI del frontend hasta la persistencia en base de datos, pasando por la fábrica, el motor de reglas, la autorización organizacional y la vinculación de la actividad con el área correspondiente.

### 4.2 Diagrama de Secuencia

```mermaid
sequenceDiagram
    actor U as Usuario
    participant Comp as RegisterNewActivity<br/>Component
    participant ActSvc as ActivityService
    participant HTTP as HttpClient +<br/>Interceptors
    participant Ctrl as ActividadComando<br/>Controlador
    participant PA as @PreAuthorize<br/>(HAS_CREATE)
    participant Man as GuardarActividad<br/>Manejador
    participant Fab as ActividadFabrica
    participant UC as GuardarActividad<br/>UseCase
    participant Auth as Autorización<br/>ContextoOrganizacional
    participant Motor as MotoresFabrica<br/>.MOTOR_ACTIVIDAD
    participant VincSvc as VincularActividad<br/>ConAreaService
    participant RepoCmd as ActividadRepositorio<br/>Comando
    participant RepoCon as ActividadRepositorio<br/>Consulta
    participant DAO as ActividadDAO /<br/>EjecucionDAO
    participant DB as PostgreSQL

    U->>Comp: Completa formulario de nueva actividad
    Comp->>ActSvc: agregarNuevaActividad(ActividadComando)
    ActSvc->>HTTP: POST /api/actividades + JSON body

    HTTP->>Ctrl: @PostMapping guardar(ActividadComando)
    Ctrl->>PA: @PreAuthorize(HAS_AREA_ADMIN_OR_ADMIN_CREATE)
    PA-->>Ctrl: Autorizado

    Ctrl->>Man: ejecutar(ActividadComando)

    Man->>Fab: construir(comando)
    Fab->>RepoCon: consultarPorIdentificador(indicadorId)
    RepoCon->>DAO: findById(indicadorId)
    DAO->>DB: SELECT indicador
    DB-->>DAO: Indicador
    Fab->>Fab: Actividad.construir(uuid, nombre, objetivo,<br/>semestre, rutaInsumos, fechaCreacion, indicador,<br/>colaboradorId, creadorId)
    Fab-->>Man: Actividad (modelo de dominio)

    Man->>Fab: construirEjecuciones(fechasProgramadas, actividad)
    Fab->>RepoCon: consultarEstadoPorNombre("PENDIENTE")
    RepoCon->>DB: SELECT estado_actividad
    DB-->>Fab: EstadoActividad(PENDIENTE)

    loop Para cada fecha programada
        Fab->>Fab: EjecucionActividad.construir(uuid, fecha,<br/>estado PENDIENTE, actividadPadre)
    end
    Fab-->>Man: List〈EjecucionActividad〉

    Man->>UC: ejecutar(actividad, ejecuciones, areaUUID, tipoArea)

    UC->>Auth: validarAccesoPorTipoArea(area, tipoArea)
    Note over Auth: Switch: DIRECCION → validarAccesoADireccion<br/>AREA → validarAccesoAArea<br/>SUBAREA → validarAccesoASubarea
    Auth->>Auth: obtenerContexto() → SecurityContextHolder
    Auth-->>UC: Acceso permitido

    UC->>Motor: ejecutar(actividad, TipoOperacion.CREAR)
    Note over Motor: MotorRegla〈Actividad〉 ejecuta<br/>reglas: validarIdentificador,<br/>validarNombre, validarObjetivo,<br/>validarSemestre, etc.

    alt Validación de reglas falla
        Motor-->>UC: throw ValorObligatorioExcepcion
        UC-->>Man: Propagación de excepción
        Man-->>Ctrl: Propagación
        Ctrl-->>HTTP: 400 Bad Request
        HTTP-->>Comp: Error
        Comp-->>U: Mensaje de error de validación
    else Validación OK
        UC->>RepoCon: existeActividadConNombreEnSemestreYArea(...)
        RepoCon->>DB: SELECT COUNT(*)
        DB-->>RepoCon: 0

        UC->>RepoCmd: guardar(actividad)
        RepoCmd->>DAO: dao.save(entidadMapeada)
        DAO->>DB: INSERT INTO actividad
        DB-->>DAO: OK
        RepoCmd-->>UC: UUID actividadGuardada

        UC->>VincSvc: ejecutar(actividad, area, tipoArea)
        VincSvc->>RepoCmd: guardarVinculacion(actividad, area)
        RepoCmd->>DB: INSERT INTO actividad_area/subarea/direccion

        loop Para cada ejecución
            UC->>Motor: MOTOR_ESTADO_ACTIVIDAD.ejecutar(estado, CREAR)
            UC->>Motor: MOTOR_EJECUCION_ACTIVIDAD.ejecutar(ejecucion, CREAR)
            UC->>UC: validarFechaProgramadaNoAnterior(fecha)
            UC->>UC: validarFechaPerteneceASemestre(fecha, semestre)
            UC->>RepoCmd: guardarEjecucion(ejecucionActividad)
            RepoCmd->>DAO: dao.save(ejecucionEntidad)
            DAO->>DB: INSERT INTO ejecucion_actividad
        end

        UC-->>Man: UUID
        Man-->>Ctrl: ComandoRespuesta〈UUID〉
        Ctrl-->>HTTP: 200 OK + {valor: UUID}
        HTTP-->>ActSvc: Observable success
        ActSvc-->>Comp: Respuesta
        Comp-->>U: Actividad creada exitosamente
    end
```

## 5. SD-04 — Consultar Actividades por Área (Flujo de Consulta/Lectura)

### 5.1 Descripción

Flujo CQRS de lectura: consulta de actividades filtradas por un área específica, con validación de autorización organizacional.

### 5.2 Diagrama de Secuencia

```mermaid
sequenceDiagram
    actor U as Usuario
    participant Comp as AreaActivities<br/>Component
    participant ActSvc as ActivityService
    participant HTTP as HttpClient +<br/>Interceptors
    participant Ctrl as ActividadConsulta<br/>Controlador
    participant PA as @PreAuthorize<br/>(HAS_GET)
    participant Man as ConsultarActividades<br/>PorAreaManejador
    participant UC as ConsultarActividades<br/>PorAreaUseCase
    participant Auth as Autorización<br/>ContextoOrganizacional
    participant AreaRepo as AreaRepositorio<br/>Consulta
    participant ActRepo as ActividadRepositorio<br/>Consulta
    participant DAO as AreaDAO /<br/>ActividadDAO
    participant DB as PostgreSQL

    U->>Comp: Navega al área de Bienestar
    Comp->>ActSvc: consultarPorArea(areaId)
    ActSvc->>HTTP: GET /api/actividades/area/{areaId}

    HTTP->>Ctrl: @GetMapping("/area/{id}")
    Ctrl->>PA: @PreAuthorize(HAS_USER_OR_AREA_ADMIN_OR_ADMIN_GET)
    PA-->>Ctrl: Autorizado
    Ctrl->>Man: ejecutar(identificadorStr)

    Man->>UC: ejecutar(identificadorStr)
    UC->>UC: UUID id = UtilUUID.textoAUUID(identificadorStr)

    UC->>Auth: validarAccesoAArea(id)
    Auth->>Auth: obtenerContexto()
    Note over Auth: Verifica: esAdminDireccion → OK<br/>esAdminArea + area coincide → OK<br/>esColaborador + area coincide → OK<br/>Otro → AuthorizationException

    UC->>AreaRepo: consultarPorIdentificador(id)
    AreaRepo->>DAO: areaDAO.findById(id)
    DAO->>DB: SELECT * FROM area WHERE id = ?
    DB-->>DAO: AreaEntidad

    alt Área no encontrada
        UC->>UC: throw ValorInvalidoExcepcion<br/>("Área no encontrada con id")
        UC-->>Ctrl: Excepción propagada
        Ctrl-->>HTTP: 400 Bad Request
    else Área encontrada
        AreaRepo-->>UC: Area

        UC->>ActRepo: consultarPorArea(area)
        ActRepo->>DAO: Consulta JPA con joins
        DAO->>DB: SELECT actividades + ejecuciones<br/>WHERE area = ?
        DB-->>DAO: List〈ActividadEntidad〉
        ActRepo->>ActRepo: Mapeo Entidad → ActividadDTO (cada una)
        ActRepo-->>UC: List〈ActividadDTO〉

        UC-->>Man: List〈ActividadDTO〉
        Man-->>Ctrl: List〈ActividadDTO〉
        Ctrl-->>HTTP: 200 OK + JSON Array
        HTTP-->>ActSvc: Observable〈ActividadDTO[]〉
        ActSvc-->>Comp: Lista de actividades
        Comp-->>U: Tabla de actividades renderizada
    end
```

## 6. SD-05 — Ciclo de Vida de Actividad (Iniciar → Finalizar/Cancelar)

### 6.1 Descripción

Flujo que cubre los cambios de estado de una actividad: **Iniciar** (PENDIENTE → INICIADA), **Finalizar** (INICIADA → FINALIZADA con registro de participantes) y **Cancelar** (INICIADA → PENDIENTE). El estado **CANCELADA** no existe en el sistema.

### 6.2 Diagrama — Iniciar Actividad

```mermaid
sequenceDiagram
    actor U as Coordinador
    participant Comp as AttendanceRecord<br/>Component
    participant ActSvc as ActivityService
    participant HTTP as HttpClient
    participant Ctrl as ActividadComando<br/>Controlador
    participant Man as IniciarActividad<br/>Manejador
    participant UC as IniciarActividad<br/>UseCase
    participant Auth as Autorización<br/>ContextoOrganizacional
    participant Motor as MotoresFabrica
    participant Repo as ActividadRepositorio<br/>Comando
    participant DB as PostgreSQL

    U->>Comp: Click "Iniciar Actividad"
    Comp->>ActSvc: iniciarActividad(ejecucionId)
    ActSvc->>HTTP: PUT /api/actividades/iniciar/{id}
```

## 9. SD-08 — Autorización por Contexto Organizacional

### 9.1 Descripción

Flujo del servicio de dominio `AutorizacionContextoOrganizacionalServicio`, que determina si el usuario autenticado tiene acceso a un recurso según su rol y posición en la jerarquía organizacional.

### 9.2 Diagrama de Secuencia

```mermaid
sequenceDiagram
    participant UC as UseCase<br/>(cualquiera)
    participant Auth as Autorización<br/>ContextoOrganizacional
    participant CtxProv as ContextoUsuario<br/>ProveedorServicio
    participant SecCtx as SecurityContext<br/>Holder
    participant AreaRepo as AreaRepositorio<br/>Consulta
    participant SubRepo as SubareaRepositorio<br/>Consulta
    participant UsrOrgRepo as UsuarioOrganización<br/>RepositorioConsulta
    participant DB as PostgreSQL

    UC->>Auth: validarAccesoAArea(areaId)
    Auth->>CtxProv: obtenerContextoActual()
    CtxProv->>SecCtx: getAuthentication().getDetails()
    SecCtx-->>CtxProv: ContextoUsuarioAutenticado<br/>{identificador, correo, rol,<br/>direccionId, areaId, subareaId}
    CtxProv-->>Auth: Contexto

    alt esAdministradorDireccion (rol == ADMIN_DIRECCION)
        Note over Auth: Acceso total a toda la dirección
        Auth-->>UC: Acceso permitido (return)
    else esAdministradorArea (rol == ADMIN_AREA)
        Auth->>Auth: areaId.equals(contexto.getAreaId())?
        alt Área coincide
            Auth-->>UC: Acceso permitido
        else Área no coincide
            Auth->>Auth: throw AuthorizationException<br/>("No tiene acceso al recurso")
        end
    else esColaborador (rol == COLABORADOR)

            Note over Auth: Acceso permitido (incluye subáreas de dicha área)
            Auth-->>UC: Acceso permitido
        else Área no coincide
            Auth->>Auth: throw AuthorizationException<br/>("No tiene acceso al recurso")
        end
    end

    Note over UC,DB: Variante: validarAccesoASubarea(subareaId)

    UC->>Auth: validarAccesoASubarea(subareaId)
    Auth->>CtxProv: obtenerContextoActual()
    CtxProv-->>Auth: Contexto

    alt esAdministradorDireccion
        Auth-->>UC: OK
    else esAdministradorArea
        Auth->>Auth: contexto.getAreaId() != null?
        alt AdminArea con área asignada
            Auth->>SubRepo: consultarPorIdentificador(subareaId)
            SubRepo->>DB: SELECT subarea + area
            DB-->>SubRepo: Subarea
            Auth->>Auth: subarea.getArea().equals(contexto.getAreaId())?
            alt Pertenece a su área
                Auth-->>UC: Acceso permitido
            else No pertenece
                Auth-->>UC: AuthorizationException
            end
        else AdminArea con solo subárea asignada
            Auth->>Auth: subareaId.equals(contexto.getSubareaId())?
            alt Subárea coincide
                Auth-->>UC: Acceso permitido (solo esta subárea)
            else No coincide
                Auth-->>UC: AuthorizationException
            end
        end
    else esColaborador
        Auth->>Auth: subareaId.equals(contexto.getSubareaId())?
        alt SubÁrea coincide
            Auth-->>UC: Acceso permitido
        else No coincide
            Auth-->>UC: AuthorizationException
        end
    end
```

```mermaid
sequenceDiagram
```
