## 1. Descripción del Proyecto

**SIBE** (Sistema de Información de Bienestar y Evangelización) es una aplicación web institucional desarrollada para la **Dirección de Bienestar y Evangelización de la Universidad Católica de Oriente (UCO)**. El sistema centraliza la planificación, ejecución y seguimiento de las actividades de la Dirección de Bienestar y Evangelización, vinculándolas con los indicadores de gestión del plan de desarrollo institucional.

### Propósito del sistema

- Gestionar la jerarquía organizacional de la Dirección de Bienestar y Evangelización (Dirección → Área → Subárea)
- Gestión de roles para el manejo de la estructura organizacional (Dirección – Área – Subárea)
- Planificar y ejecutar actividades con el registro de asistencia a través del carnet universitario y el documento de identidad
- Registrar participantes (estudiantes, empleados y externos) con sus perfiles completos
- Medir el impacto de las actividades a través de indicadores vinculados a proyectos y acciones del PDI
- Generar reportes y evidencia para procesos de Acreditación, Registro Calificado, ISO 9001 y Ministerio de Educación Nacional

### Stack tecnológico principal

| Capa | Tecnología |
|------|-----------|
| Backend | Java 17 · Spring Boot 3.5.0 · Spring Security (JWT stateless) · Spring Data JPA |
| Frontend | Angular 16.2.0 · TypeScript 5.1.3 · Angular CLI 16.2.13 |
| Base de datos | PostgreSQL (`sibe_db2`, puerto 5432) |
| Construcción | Gradle 8.13 |
| Arquitectura | Hexagonal + CQRS (Dominio / Aplicación / Infraestructura) |
| Seguridad | JWT Bearer, BCrypt, roles CRUD con bits de permiso |

## 12. Convenciones de Nomenclatura

### Nomenclatura de archivos

Todos los artefactos siguen el patrón:

```
{número}.{nombre_en_minuscula_con_guion_bajo}.md
```

Ejemplos: `1.vision.md`, `24.diagrama_del_modelo_entidad_relacion.md`, `41.documento_tecnico_final.md`

### Estructura interna de cada artefacto

Cada artefacto de esta colección sigue la siguiente estructura estándar:

```markdown
# Artefacto N — Nombre del Artefacto

| Campo     | Detalle |
|-----------|---------|
| Proyecto  | SIBE    |
| Versión   | x.y.z   |
| Fecha     | YYYY-MM-DD |
| ...       | ...     |

---

## Tabla de Contenidos
## 1. Introducción / Propósito
## 2. [Contenido específico del artefacto]
...
## N. Historial de Cambios
```
